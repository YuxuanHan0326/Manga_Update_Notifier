# 安全策略（Security Policy）

## 支持范围
当前仅保证 `main` 分支最新版本的安全修复维护。

## 漏洞上报
如果你发现安全问题，请不要公开提 issue 细节。建议通过以下方式私下联系维护者并提供：
- 影响范围（组件/接口）
- 复现步骤
- 潜在影响
- 建议修复方向（可选）

## 安全部署基线
本项目是 NAS 自托管服务，默认不内建重型认证。生产部署建议：
- 使用反向代理并启用鉴权（Basic Auth/SSO 至少一种）
- 全程 HTTPS
- 尽量只内网访问或通过 VPN 暴露
- 限制 `/api/*` 的外网暴露范围

## 密钥与配置
- 严禁将真实密钥、令牌、Webhook URL 提交到仓库。
- 通过环境变量注入敏感配置。
- 提交前确认 `.env`、数据库、日志未进入版本控制。

## 依赖与镜像安全
- CI 内置依赖审计（`pip-audit`、`pnpm audit --prod`）与镜像扫描（Trivy）。
- 对 `HIGH/CRITICAL` 级别问题应优先修复后再发布。
